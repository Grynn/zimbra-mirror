Creating or Deleting Classes of Service

Class of Service (COS) determines what default attributes a mail account has and what features are added or denied. You can create different COS types to manage what features various user types can have. Before you create the COS for your mail system, consider the types of end users and what type of features they may need.

There is no limit to the number of COSs, you can create. A COS is a global object and is not restricted to a particular domain or set of domains.

To create a new class of service:

  1. Click Class of Service in the navigation pane.

  2. Click New, from the toolbar.

  3. In the General Information tab, enter a COS name and a description of the type of COS. The name cannot contain any spaces. You can use the Notes area to keep a record of changes to the COS.

  4. In the Features tab, deselect any features you do not want included in the COS.

  5. In the Preferences tab, define the initial preferences for how messages will be saved and viewed.

    • Users can change the preferences from the Zimbra Web Client, Options tool.

  6. In the Themes tab, select which themes should be available in user accounts. In order for themes to be used, the Change UI themes feature must be enabled in the Features tab.  

  7. In the Zimlets tab, select which Zimlets should be available in users accounts

  8. In the Server Pool tab, select which servers to include in the pool. The servers in the pool are randomly assigned new accounts.

    • Note: When you create an account, you can override this action and assign the account to a specific mail server.

  9. In the Advanced tab, set the advanced settings including quotas, password constraints, session length, and message lifetimes.

    The Advanced tab is also where you can disable attachment viewing.

  10. Click Save.

You can edit a COS at any time.

To delete a class of service:

  1. Click Class of Service, in the navigation pane.

  2. Click the COS, to be deleted from the contents pane.

  3. Click Delete.

All accounts associated with the deleted COS are automatically reassigned to the default COS.
