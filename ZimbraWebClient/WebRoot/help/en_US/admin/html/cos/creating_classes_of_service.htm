Creating Classes of Service

Class of Service (COS) determines which default attributes a mail account has and which features are added or denied. You can create different COS types to manage features different user types can have. Before you create the COS for your mail system, consider the types of end users and what type of features they might need.

There is no limit to the number of COSs you can create. A COS is a global object and is not restricted to a particular domain or set of domains.

Procedure

  1. Go to Configure>Class of Service.

  2. In the gear icon select New.

  3. In the General Information page, enter the COS display name and a description of the type of COS. The display name cannot contain any spaces. You can use the Notes area to keep a record of changes to the COS.

  4. In the Features page all the features available for the COS are displayed. Select the  features to include in the COS.

  5. In the Preferences page, define the initial preferences for how messages are saved and viewed. You can also select a specific locale for the ZWC view. If a locale is not specified, the browser locale is the default.

    • Users can change the preferences from the Zimbra Web Client.

  6. In the Themes page, select which themes should be available in user accounts. In order for themes to be used, the Change UI themes feature must be enabled in the Features page.  Note: If you limit the themes, one of the themes you select must be listed as the default theme.

  7. In the Zimlets page all Zimlets that have been deployed are listed. You can select which Zimlets should be mandatory for all users, which are enabled and users can manage usage from their Zimlets Preferences page, and which Zimlets should be disabled.  Zimlets that are marked as mandatory are not listed in the user's Zimlets Preferences page.

  8. In the Server Pool page, select which servers to include in the pool. The servers in the pool are randomly assigned new accounts.

    • When you create an account, you can override this action and assign the account to a specific mail server.

  9. In the Advanced page, set the advanced settings including quotas, password constraints, session length, and message lifetimes. The Advanced page is also where you can disable attachment viewing.

  10. Click Save.

You can edit a COS at any time.
