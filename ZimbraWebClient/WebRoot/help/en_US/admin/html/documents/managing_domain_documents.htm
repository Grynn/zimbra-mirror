Creating Domain Documents Account  

One domain Documents account can be created per domain. The Documents wizard is used to quickly create the account and set the permissions for accessing the account. When the domain Documents account is created, the account includes a Notebook folder and a Template Notebook folder.

The Notebook folder can be used to create, collect, and organize information that can be shared with others. The default domain Documents access is set to allow all users in the domain to view, edit, add, and delete the contents of the Domain Notebook folder.  You can modify these permissions.

The Template Notebook does not contain any template files. The Global Documents Template Notebook folder is used as the base template files for all domains and user notebooks folders. Note: Changing templates or creating new templates is not available at this time.

  • Once you create the domain Documents account, you should not delete the account. If the Documents Notebook should not be accessed, remove the access privileges.

How to create a domain Documents account

  1. From Configuration>Domains, select the domain and click Edit to open the domain's configuration tabs.

  2. Click Create Documents on the toolbar. The Document Creation Wizard opens.

  3. An account for the domain Documents must be created. Enter the Documents account name and the account password.  The name must be one word with no spaces.

  4. Click Next.  The Global Access Control dialog displays. If you want to share the domain Documents Notebook with others, set the global access permissions for the following:

    • All users in this domain.  The default is all users in the domain can View, Edit, Delete and Add pages to the domain Notebook.

    • All users in all domains. All users in all domains in the Zimbra Collaboration Suite environment can be given access. You can enable access to View, Edit, Delete, and Add.

    • Public.  The public can be given view-only access.

  5. Click Next. The Advanced Access Control dialog displays. Here you can give access to one or more distribution lists and to individual accounts. Click Next and then Finish.

  6. Notify users that they have access to the domain Documents Notebook folder.

When users open Documents from their account, they will need to right-click on Notebooks, select Link to Shared Notebook and enter the domain Documents account email address and in the Path field type the name of the folder.  The default name is Notebook.

To access the domain Documents Notebook, log on to the domain Documents account.  See the Zimbra Web Client Help for how to create pages within the notebook.
