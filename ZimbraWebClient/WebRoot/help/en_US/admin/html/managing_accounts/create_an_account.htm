Provisioning  Accounts

When you provision user accounts, the mailbox is created on the designated server and a directory account is created on the LDAP server.

Each account is assigned a Class of Service (COS) that defines features and preferences for the mailbox, but you can override the COS attributes when you configure the account.  When you initially create an account, you need to consider how you want to set up the account and whether the configuration should be changed from the COS configuration.

  • What display name should appear in the user's messages?

  • What features and preferences should be set for the account?

  • Will alias and forwarding addresses be required?

  • What password restrictions to set?

  • Can the account receive mail attachments?

  • What should the account disk space quota be set to?

  • Is there a set lifetime for messages?

  • How long should the mailbox session stay open?

  • Should the account name be hidden in the Global Address List?

  • Do you need to set a default time zone for the account that is different than yours?

Important:  IMAP and POP client users may not have the restrictions that are configured in their COS or Account settings.

To create an account:

  1. Click Accounts, in the navigation pane.

  2. In the Accounts toolbar, click New.

  3. The New Account Wizard appears.  The wizard steps you through a series of dialog boxes to set up contact information, features, mail aliases, forwarding information, mailbox preferences, and advanced features.

    • General Information. The minimum account configuration requires completing the fields marked with an asterisk in General Information. You must enter the Account name, select the host name, and enter the Last name. If auto is checked next to the Display name field, the last name is automatically added as the display name. If you enter a canonical address, this address can be displayed in email messages from the account. If you do not want this account displayed in the Global Address List (GAL), check Hide in GAL.

    • An account name cannot have spaces or use symbols. You can type a period (.) between words. Example: john.smith. This becomes the name in the email address, i.e. john.smith@example.com

In the Account Setup section, if the account should not be immediately available, you can change the account status from Active.  If auto is checked next to the Class of Service field, the account is assigned the domain COS. If no domain COS exists, the default COS is automatically assigned. Uncheck the auto box to select another COS for the account.

You can select the password for this account and require the user to change the password the first time he logs on.

The Notes section can be used to add a description of this account and to keep notes about the account. The description is displayed with the account information in the account content pane; notes are not displayed.

    • Contact Information. Phone number, company name, department name, and address information can be added.

    • Aliases. If the account should have aliases, enter email aliases for this account.

    • Forwarding. Forwarding is enabled by default.

    • Features, Preferences, Themes, Zimlets, and Advanced. The default feature, preferences, and advanced features configuration is based on the COS assigned to the account.  You can changes these for individual accounts. A Reset to COS value link is displayed when you make a change to let you know that the account configuration overrides the COS configuration.

  1. Click Finish to save.

  • Note: If you need to create large numbers of accounts at once, an alternate method for creating accounts is to use the command-line tool called zmprov. For more information on using this tool for bulk provisioning, see the Administrator Guide.
