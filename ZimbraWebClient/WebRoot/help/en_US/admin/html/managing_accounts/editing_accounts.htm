Editing Accounts

The COS sets the default features, preferences and general usage for accounts.  You can edit an account to override the COS.

  1. Go to Manage >Accounts.

  2. Select the account to edit and in gear icon click Edit.

  3. Edit any of the information on the pages listed in the Navigation pane..

    • General Information page displays account information, password, and status. You can modify any of these.  

      • If you change the account name, you change the email address for that account. You may need to create an alias with the old account name so that messages are redirected to the new account name.

    • Contact Information page displays telephone and company name and address fields.

    • Member Of page displays which distribution lists the account is a member of. You can add distribution lists to the account and remove the account from distribution lists.

    • Features page lists all the features that this account can use. If you change a feature on this page, a Reset to COS value notice displays to let you know that this account feature does not match the COS feature configuration.

    • Preferences page lists how users' mailboxes behave when a message is composed or received. Users can modify their preferences from Zimbra Web Client Options.  If you change a preference on this page, a Reset to COS value notice displays to let you know that this account does not match the COS.

      • The account's Preferences page is updated to show changes users make from Zimbra Web Client.  

    • Aliases page is where alias addresses are added. An email alias is an email address that redirects all mail to this account. An alias is not an email account. An unlimited number of aliases can be created for an account.

    • Forwarding page is where you enable the account to be able to set up a forwarding address. You can enter a forwarding address and .users can change this address from Zimbra Web Client Preferences tab. You can also define a forwarding address that is hidden from the user. When email is received in the account, a copy is sent to the forwarding addresses you set up. See Forwarding Mail.

    • Free/Busy Interop page configures the foreign principal for the account. This is necessary when ZCS is deployed in a mix of Zimbra servers and third party email servers and Calendar is an important feature with your users. This sets up  free/busy scheduling across the mix so that users can efficiently schedule meetings. For this configuration, the cn setting in the legacyExchangeDn attribute corresponds to the zimbraForeignPrincipal attribute. This sets up a mapping from the Zimbra account to the corresponding object in the AD.  

    • Themes page is where the available and installed themes are listed.  See Defining Themes for the Zimbra Web Client UI.

    • Zimlets page lists the Zimlets that are available.

    • Advanced page is used to set more advanced features such as attachment setting policy, quotas, password policy, failed login policy for the account.

    • ACL and Admin View page are used if the account is a delegated admin account. See Delegated Administration.

  1. Click Save.
