Prevent Users from Changing Passwords

Normally, users can change their own passwords whenever they want.  If you enforce password changes, users are forced to change the password on a regular basis. However, if you are using External Active Directory or External LDAP for authentication, you must prevent users from changing their passwords.  

You can set Password locked to prevent passwords from being changed either at the COS level or per account.

To lock passwords for all users in a Class of Service:

  1. Select Class of Service from the navigation pane.

  2. Select the COS to change from the contents pane.

  3. Click the Advanced tab.

  4. Check the Password Locked box.

  5. Click Save to commit your changes.

To prevent individual accounts from changing their passwords:

  1. Select Accounts from the navigation pane, choose the domain containing the account.

  2. Select the account from the list on the contents pane.

  3. Click the Advanced tab.

  4. In the Password Locked field check the box next to the setting.

  5. Click Save to commit your changes.
