Global Settings to Block Mail Attachments

The Attachments tab can be configured with global rules to reject mail with files attached , and to disable viewing files attached to mail messages in users’ mailboxes. When attachment settings are configured in Global Settings, the global rule takes precedence over COS and Account settings.

The attachment settings are as follows:

  • Attachments cannot be viewed regardless of COS. Users cannot view any attachments. This global setting can be set to prevent a virus outbreak from attachments, as no mail attachments can be opened.

  • Attachments are viewed according to COS. This global settings states the COS sets the rules for how email attachments are viewed.

Selecting extensions to block

Select which file types cannot be viewed. Messages with attachments with these extensions are  rejected and the sender gets a bounce notice. The recipient does not get the mail message and is not notified.  If you want the recipient to receive notification, you can enable Send blocked extension notification to recipient.

Note: Attachments settings can also be set for a Class of Service (COS) and for accounts.

Messages with those type of files attached are rejected and the sender gets a bounce notice. The recipient does not get the mail message and is not notified.  If you want the recipient to receive notification, you can enable Send blocked extension notification to recipient.

If attachment blocking is not enough

  • To temporarily block message delivery to an account, you can change the account status to "maintenance".

  • To temporarily block message delivery for all accounts on a particular Zimbra mail server, you can stop mailbox service on that host. Mail will be queued at the MTA until mailbox service is resumed.
