Making Free/Busy Schedule Available

When ZCS is deployed in a mix of ZCS servers and third-party email servers and Calendar is an important function with your users, you may want users to be able to see the free/busy scheduling across the mix so that users can efficiently schedule their meetings.  

ZCS can query the free/busy schedules of users on Microsoft Exchange 2003/2007 servers and also can propagate the free/busy schedules of ZCS users to the Exchange server. To set this up, you configure Global Config or Domain, Class of Service, and Account settings.

To set Free/Busy interoperability up from the administration console, configure the following:

  • Either globally or by domain configure the Exchange server settings in the Interop tab.

  • In the global settings or domain Interop tab or the Class of Service (COS) Advanced tab, add the o and ou values that are in the legacyExchangeDN attribute for Exchange. These values correspond to the ZCS domain attribute zimbraFreebusyExchangeUserOrg.

  • In the Accounts Free/Busy Interop tab, configure the foreign principal for the account. The cn value in the legacyExchangeDn attribute corresponds to the zimbraForeignPrincipal attribute. This sets up a mapping from the ZCS account to the corresponding object in the AD.

  • To find these values on the Exchange server, you can run the Exchange ADSI Edit tool and search the legacyExchangeDN attribute for o= , ou= , and cn= values.

Global Config Settings

The Interop tap is used to configure the Exchange server settings.

You configure the following on the tab:

  • Exchange server URL.  This is the Web interface to the Exchange.

  • Exchange authentication scheme, either Basic or Form.

    • Basic is authentication to Exchange via HTTP basic authentication.

    • Form is authentication to Exchange as HTML form based authentication.

  • Exchange user name and password. This is the name of the account in Active Directory and password that has access to the public folders.

Domain Setting

The Interop tab is used to configure the Exchange server settings. This information is the basic property of the Exchange server account that has access to the Exchange public folders. In order to enable the Exchange free/busy interop implementation, a  single AD must be in the system or the Global Catalog must be available and the ZCS server must be able to access the HTTP(S) port of IIS on at least one of the Exchange servers.

You configure the following on the tab:

  • Exchange server URL.  This is the Web interface to the Exchange public folders.

  • Exchange authorization schema, either Basic or Form.

    • Basic is authentication to Exchange via HTTP basic authentication.

    • Form is authentication to Exchange as HTML form based authentication.

  • Exchange user name and password . This is the name of the account  and password that has access to the public folders.

  • Domain settings overwrite Global settings.

COS Setting

The Free/Busy Interop section of the Advanced tab is used to set the O and OU used in legacyExchangeDN attribute (i.e. legacyExchangeDN : /o=First Organization/ou=First Administrative Group).

Account Settings

Two tabs have Free/Busy Interop configuration options.

  • Advanced tab. This is the same as the COS Setting configuration unless you want to change it for a specific account.

  • Interop tab. This is the tab to configure the foreign principal for the account.  This is a mapping from the ZCS account to the corresponding object in the Active Directory. The Zimbra foreign principal maps like this: legacyExchangeDN : /o=.../ou=.../cn-Recipient/cn=<Zimbra foreign principal>. The Zimbra foreign principal is usually the user's address.

This is a multi-value attribute, you can add more than one account address. A prefix is added to the address.  For Exchange the prefix is AD.
