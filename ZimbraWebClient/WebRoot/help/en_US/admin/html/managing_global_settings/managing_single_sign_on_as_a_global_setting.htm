Managing Single Sign On as a Global Setting

When ZCS is configured for single sign-on authentication to the Zimbra Web Client, the server is configured to redirect users to a URL for single sign in. You can configure the web client sign in and sign out URL and the UAs allowed for the sign in URL.

Procedure

  1. Go to Configure > Global Settings, Authentication page.

  1. In the Web client login redirect URL text box, type the URL to redirect users to when the auth token is expired. Enter as https//example.com.

  2. In the Web client logout redirect URL text box, type the URL to redirect users to when they click Sign out. Enter as https//example.com.

  3. In the Allowed user agent when login text box, type the supported platforms and browsers that are allowed. If you do not set this, all user agents (UA) are allowed. Click Add to add more than one  type.

  4. In the Allowed user agent when logout text box, type the sign out URL and UAs allowed to redirect users to when they sign out. Click Add to add more than one type.

  5. In Allowed IP when login text box, type the IP addresses that are allowed as the sign in URL. Click Add to add more than one  type.

  6. In Allowed IP when logout text box, type the IP addresses that are allowed as the sign out URL. Click Add to add more than one  type.

  7. Click Save.

 

Example of how to enter text to describe the user agent if you wanted to limit access to only Firefox, Internet Explorer, Chrome, and Safari on computers running Windows, and Safari on Apple Mac Computers. Enter each in a separate text box.

.*Windows.*Firefox/3

.*MSIE.*Windows.*

.*Windows.*Chrome.*

.*Windows.*Safari.*

.*Macintosh.*Safari.*
