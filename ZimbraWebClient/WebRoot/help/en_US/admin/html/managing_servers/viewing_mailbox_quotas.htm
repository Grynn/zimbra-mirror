Viewing Mailbox Quotas

Account quota is the storage limit allowed for an account. Email messages, address books, calendars, tasks, Documents notebook pages, and Briefcase files contribute to the quota. When the quota is reached, the message is sent to the deferred queue to be delivered when the mailbox has space.  The bounce queue lifetime is set for five days. The deferred queue tries to deliver a message until this bounce queue lifetime is reached before bouncing the message back to the sender.

Users will need to delete items off the server to get below their quota limit, or you can increase their quota.

You can check account mailbox quotas from Server Statistics. The Mailbox Quota displays the following information for each account sorted by mailbox size in descending order.  You can click on the Quota, Mailbox Size, or Quota Used headers to change the viewing order.

  • The Quota column shows the mailbox quota allocated to the account. Quotas are configured either in the COS or by Account.

  • The Mailbox Size column shows the disk space used.

  • The Quota Used column shows what percentage of quota is used.

If a quota is not set, the Quota column displays 00.0MB. The account has unlimited use of disk space.
