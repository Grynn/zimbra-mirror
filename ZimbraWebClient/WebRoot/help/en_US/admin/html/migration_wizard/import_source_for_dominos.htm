Import Source for Lotus Domino

Enter the information required to connect and authenticate to the Domino server. If the migration wizard can determine the Domino server name, administration ID file path, and Notes INI file path, The Import Source information is displayed in the dialog. If this information is missing, you must enter it in order to connect and authenticate to the Domino server.

To migrate user’s Lotus Domino private folders, you must check Migrate Private Folders and complete the following:

  1. Specify the path to the directory containing ID files of all the Lotus Domino users to be migrated. The names of the ID files should follow the format <user shortname>.id.

  2. Specify the path to the file containing passwords for these ID files. The users and passwords should be specified in format <user full name>:<password>. The user full name format is < username>/[<organization unit>/]<organization>. For example, if the user is Joe, the organization is Zimbra and the organization unit is marketing, than the user full name is joe/marketing/ zimbra.
