Installing Certificates

You can use the Certificate Installation Wizard to

  • Install self-signed certificate

  • Generate the Certificate Signing Request ( CSR) that is sent a commercial certificate authorizer

  • Install the commercially signed certificate when it has been authorized.

See Generating Certificates

When ZCS is installed, the self-signed certificate is automatically installed and can be used for testing ZCS. You should generate the CSR and install the commercial certificate when ZCS is used in your production environment.  

When you are using a commercially signed certificate, the Certificate Install Wizard is run twice:

  • To generate the CSR and download so you can send it to a Certificate Authority who issues a signed certificate.

  • To install the signed certificate when it is returned.

How to generate a Certificate Signing Request

To generate the CSR, you complete a form with details about  the domain, company, and country and then generate a CSR with the RSA private key. You save this to your computer and submit it to your commercial certificate authorizer.

You will be required to enter information about your company. Before you generate the CSR, you must know the following details

  • Common Name ( CN). This is the exact domain name that should be used to access your Web site securely.

  • Are you going to use a wildcard common name?  If you want to manage multiple sub domains on a single domain on the server with a single certificate, check this box. An asterisk (*) is added to the Common Name field.

  • Country Name (C) The country you want the certificate to display as your company location.

  • State/Province ( ST) The state/province you want the certificate to display as your company location.

  • City ( L) The city you want the certificate to display as your company location.

  • Organization Name (O). Your company name.

  • Organization Unit ( OU). The unit name (if applicable).

  • Are you going to use Subject Alternative Name (SAN)? The input should be a valid domain name. When SAN is used the domain name is compared with the common name and then to the SAN to find a match. You can create multiple SANs. When the alternate name is entered here, the client ignores the common name and tries to match the server name to one of the SAN names.

To generate the CSR

  1. Select the target server. Select All Servers if you are going to deploy  the certificate to all the servers.

  2. Select the server for the certificate. Click Next.

  3. Select Generate the CSR for the commercial certificate authorizer. Click Next.

  4. In the Generate the Certificate Signing Request dialog, enter the server information as described above.  Click Next. The CSR including the RSA Private Key is generated.

  5. Click Download the CSR and save the file to your computer. Change the default name to your servername.csr.

  6. After you save the . csr name, on the Certificate Installation Wizard dialog, click Finish.

Now you submit the . csr file to a commercial certificate authorizer. When they send the digitally signed certificate back, use the Certificates Wizard to install the certificate on the ZCS.

To install the commercial certificate

When the commercial certificate is received, save the signed certificate to a folder that you can access from the ZCS administration console. Then start the Certificate Installation Wizard.

  1. Select the server that the commercial certificate was created for. Click Next.

  2. Select Install the commercially signed certificate. Click Next.

  3. Verify that the information displayed is the correct information for the CSR. Click Next.

  4. Click Browse to find the commercial certificate. Click Next.

  5. The target server is listed. Click Install.

  6. When the certificate is installed, you must restart the server to apply the certificate. Restart the server from the command line. Click zmcontrol stop and then   zmcontrol start.
