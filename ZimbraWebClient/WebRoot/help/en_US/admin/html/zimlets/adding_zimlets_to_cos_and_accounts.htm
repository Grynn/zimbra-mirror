Adding Zimlets to COS and Accounts

Several Zimlets are deployed with ZCS and are automatically enabled in the default Class of Service (COS). New Zimlets that you deploy are added to the Zimlets tabs in Class of Service and Accounts configuration, but they are not enabled.  You must go to the COS or to the individual accounts and enable the new Zimlets.

To enable Zimlets by COS

  1. Select the COS and click Edit.

  2. Go to the COS Zimlets tab and check the Zimlets to be used

  3. For the checked Zimlets, set the one of the following:

    • mandatory, if the Zimlet cannot be enabled or disabled from an account's Preference pages.

    • enabled, if the Zimlet should be on by default. Users can disable the Zimlet.

    • disabled, if the Zimlet should be off by default.  Users can enable the Zimlet.

  4. Save your changes.

To enable Zimlets from an account

  1. Select the account to change and click Edit.

  2. Go to the Zimlets tab. To change which Zimlets are enabled for this account from the COS, click Limit the Zimlets available to this user to:.

  3. Check the Zimlets for the accounts and select one of the following:

    • mandatory if the Zimlet cannot be enabled or disabled from an account's Preference pages.

    • enabled if the Zimlet should be on by default. Users can disable the Zimlet.

    • disabled if the Zimlet should be off by default.  Users can enable the Zimlet.

  4. Save your changes.
