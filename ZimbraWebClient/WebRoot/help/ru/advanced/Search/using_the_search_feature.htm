Использование поиска

Поиск позволяет находить сообщения, контакты, встречи, страницы документов и файлы.  Поиск можно выполнять по определенным словам, датам, времени, URL-адресу, размеру, тегу, по статусу прочитанного или непрочитанного сообщения, по наличию или отсутствию вложенных файлов, по определенному типу вложений и по другим признакам.

@CollaborationProductFullName@ предлагает два инструмента поиска.

  • Поиск. Это быстрый поиск выполняемый по запросу поиска, отображаемому в текущий момент в текстовом поле поиска. В поле поиска можно ввести одно из ключевых слов поиска, после чего автоматически отобразится список возможных аргументов поиска. Кроме того, стрелка раскрывающегося списка слева от поля поиска позволяет выбирать тип элементов для поиска. Поиск можно выполнять в сообщениях, контактах, в том числе в списках контактов организации, задачах, встречах, на страницах и в файлах. Это быстрые профили поиска, описанные в разделе Настройки быстрого поиска.

  • Расширенный поиск открывает новую область и упрощает выполнение более сложного поиска.  Запросы расширенного поиска можно сохранить и повторно использовать их в будущем.

Поиск сообщений

Если вы знакомы с поиском с условием соответствия тексту или с функцией «Найти и заменить» в приложении Microsoft Word, обратите внимание, что поиск содержимого в веб-клиенте @ClientNameLong@ немного отличается от процедуры сопоставления символьных строк.

Синтаксис поиска @CollaborationProductShortName@ описан ниже.

  • Можно выполнять поиск фраз, но каждое слово фразы фактически соответствует только одному слову. Варианты орфографии недопустимы. Например, если выполняется поиск слова яблоки, сообщения со словом яблоко не являются соответствием. Можно выполнять поиск по имени домена, включая «.» (точку).

  • При поиске не учитывается регистр букв: фразы Север, север и СЕВЕР — равнозначны.

  • В тексте поиска нельзя использовать следующие специальные знаки. ~ ' ! # $ % ^ & * () _- + ? / { }[ ] ; : "

  • Поддерживается использование звездочки (*) в качестве шаблона. Таким образом, поиск строки «ты*» возвращает элементы со словами «тыква», «тысяча» и т. д.

  • Поиск содержимого выполняется в теле сообщения, а также в любом файловом вложении (читаемом системой), которое оно может иметь. Читаемое системой вложение представляет собой тип файла, который можно преобразовать в формат HTML. В том числе документы Microsoft Office (Word, PowerPoint или Excel), а также текстовые файлы, исключая изображения и аудиофайлы.
