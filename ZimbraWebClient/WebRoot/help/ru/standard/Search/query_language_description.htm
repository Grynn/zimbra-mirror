Описание языка запросов

Этот расширенный раздел подробно описывает грамматику языка поиска, используемую функцией поиска клиента @CompanyName@. Для получения вида, отображаемого в интерфейсе, всегда применяется определенный тип запроса.

Структура языка поиска

Простой поиск можно выполнить, просто введя слово в поле поиска. Только слова (слова без оператора поиска) используются для поиска в операторе «content:», что соответствует любому тексту в сообщении.

Расширенный поиск можно выполнить, указав оператор поиска. Оператор поиска — это специальное ключевое слово, за которым следует двоеточие, а затем другие параметры, специфические для этого оператора. Например:

  • in: входящие оператором является «in», а параметром — «входящие», что возвращает сообщения, которые содержатся в папке с именем «Входящие».

  • оператором в from:petrov является «от», а параметром «someone», что выдает сообщение, которые содержат слово «someone» в электронном адресе.

Перед любым ключевым словом можно поставить префикс «not» (НЕ), чтобы определить элементы, которые не содержат критерий, например not in: входящие. В профиле поиска не учитывается регистр букв; это значит, что «in:входящие» равнозначно «in:Входящие». Знак минуса (-) является синонимом NOT. Поэтому not in:входящие равнозначно -in:входящие.

В большинстве случаев нет необходимости включать знаки пунктуации, строку поиска, поскольку они игнорируются кодом поиска. Но в определенных случаях это не так (например, поиск времени «9:30» в сообщении), и в этих случаях параметр поиска следует заключать в кавычки. Например: subject:«9:30» возвращает сообщения, которые содержат в теме «9:30».

Допустимые знаки в параметре поиска.

  • Следующие знаки можно использовать в параметре поиска только заключенными в кавычки: ~ ' ! # $ % ^ & * ( ) _ ? / { }[ ] ; :

  • Следующие знаки можно использовать в параметре поиска, только если они не являются первыми знаками в строке: - + < >

Несколько условий поиска

Если указываются несколько условий поиска (разделенные пробелами), то по умолчанию они объединяются оператором «AND». in:Входящие Петр означает «выдать сообщения, которые находятся в папке «Входящие» И содержат слово «Петр». Для поиска с использованием нескольких критериев, можно найти элементы соответствующие одному или всем указанным критериям. Можно выполнить оба типа поиска с использованием функции создания расширенного поиска.

Для всех областей поиска, кроме обычного поиска, правила следующие.

  • Поиск сообщений, соответствующих любым указанным критериям называется поиском с условием 'OR', потому что, если сообщение содержит X или Y, оно считается соответствующим условиям поиска. Для панелей расширенного поиска с несколькими установленными флажками выбор нескольких вариантов на одной панели создает поиск для этих элементов с условием OR.

  • Поиск сообщений, которые содержат X и Y называется поиском с условием AND, потому что сообщение должно соответствовать указанным критериям, чтобы считаться соответствием. Для панелей расширенного сервера с установленными флажками, открытие нескольких копий одной и той же панели и установка разных флажков на каждой из них делает необходимым определение критериев поиска как поиска с условием AND.

В запросе появится только «OR». Если задан параметр показывать запрос поиска на панели поиска по мере выбора в поле «Расширенный поиск», текстовое окно поиска будет обновляться и отображать полученный запрос. В профиле поиска типа AND слово AND не отображается.

Совет. Использование скобок с операторами AND и OR. Слова в скобках считаются единицей. Например, from: (иван сидоров) равнозначно from:иван AND from: сидоров. Если использовать OR в скобках, from:(иван or иванов), результат поиска будет from:иван OR from: сидоров.

Использование * в качестве подстановочного знака в поиске

Звездочку (*) можно использовать в качестве подстановочного знака при поиске содержимого, которое содержит слова с похожим написанием.

Используйте звездочку * как подстановочный знак. Например, строка поиска ты* выдает элементы ты, тыква, тысяча и т.д.

Примеры и описания ключевых слов

content:

Определяет текст, который должен содержаться в сообщении. Например, content:яблоки находит все элементы, которые содержат слово «яблоки».

from:

Определяет имя или электронный адрес отправителя, которые содержатся в заголовке «От». Это может быть текст, например «Иван Иванов III», адрес электронной почты, например «ivan@acme.com» или домен, например «zimbra.com».

to:

Аналогично условию «from:», за исключением того, что здесь определяется один из получателей, указанных в заголовке «Кому».

cc:

Аналогично условию «from:», за исключением того, что здесь определяется получатель из заголовка сообщения «Копия».

subject:

Определяет текст, отображаемый в заголовке сообщения «Тема». Примером может быть subject:новая политика отпусков.

in:

Определяет папку. Например, in:отправленные выдаст все элементы в папке Отправленные.

has:

Определяет атрибут, который должен содержаться в сообщении. Типы объектов, которые можно определить: «attachment» (вложение), «phone» (телефон) или « url» (URL-адрес). Например, если указать has:attachment будут найдены все сообщения, которые содержат одно или несколько вложений любого типа.

filename:

Определяет имя файла вложения. Например, в результаты поиска filename: query.txt будут включены сообщения с вложенным файлом с именем « query.txt».

type:

Определяет поиск во вложениях определенного типа. Типы вложений, которые можно определить: «text», «word», «excel» и «pdf». Например, если указать type:word «hello», будут найдены сообщения с вложениями формата Microsoft Word, содержащие слово «hello».

attachment:

Определяет элемент с определенным типом вложения. Например, если указать attachment:word, будут найдены все сообщения с вложениями формата Word.

is:

Поиск сообщений с определенным статусом. Допустимые значения «unread» (непрочитанное), «read» (прочитанное), «flagged» (с флагом), «unflagged» (без флага), «sent» (отправленное), «draft» (черновик), «received» (полученное), «replied» (отвеченное), «unreplied» (неотвеченное), «forwarded» (пересланное), «unforwarded» (непересланное), «anywhere» (в любой папке), «remote» (в общей папке), «local» (локальное), «sent» (отправленное). Например, is: unread найдет все непрочитанные сообщения.

date:

Это ключевое слово используется для определения даты с использованием формата, принятого по умолчанию в используемом браузере (для русского языка формат следующий: дд.мм.гггг). Например, если указать date:01.02.2010, будут найдены сообщения, датированные 1 февраля 2010 года. Можно использовать знаки больше (>) или меньше (<), обозначающие после или до. >= и <= также допустимы.

after:

Определяет сообщения отправленные после определенной даты. Например, after:01.02.2010 определяет сообщения, отправленные после 1 февраля 2010 года.

before:

Аналогично условию after:, но определяет сообщения, отправленные до указанной даты.

size:

Определяет сообщения, общий размер которых, включая вложения, соответствует определенному объему в байтах, килобайтах или мегабайтах. Например, если указать size:12 kb, то будут найдены сообщения, размер которых составляет точно 12 КБ. Вместо слов «больше» или «меньше» можно использовать знаки больше (>) или меньше (<).

larger:

Аналогично условию «size:», но определяет сообщения, размер которых превышает указанный.

smaller:

Аналогично условию «size:», но определяет сообщения, размер которых меньше указанного.

solo:

Находит почтовые сообщения, которые еще не были отвечены.

tag:

Находит сообщения, помеченные определенным тэгом. Например, если указать tag:amber, будут найдены сообщения с тегом, который называется «Амбра».

priority:

Находит сообщения, задачи или элементы ежедневника с высоким («high») или низким («low») приоритетом. Например, priority:high найдет элементы с высоким приоритетом.
